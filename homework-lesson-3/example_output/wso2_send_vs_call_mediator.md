# Відмінність медіаторів Send та Call у WSO2

## Вступ
У WSO2 існує кілька типів медіаторів, які використовуються для обробки повідомлень. Два з найбільш поширених медіаторів — це **Send** та **Call**. Хоча вони виконують подібні функції, між ними є суттєві відмінності.

## Send Mediator
- **Призначення**: Send Mediator використовується для відправки повідомлень до визначеного кінцевого пункту (endpoint). Це може бути як статичний, так і динамічно визначений кінцевий пункт.
- **Синтаксис**: 
  ```xml
  <send>
      <endpoint>
          <address uri="http://example.com/service"/>
      </endpoint>
  </send>
  ```
- **Потік**: Send Mediator виводить з потоку (OUT sequence) або входить у потік (IN sequence) і не повертається назад. Після виконання Send Mediator подальші медіатори не будуть оброблені.
- **Використання**: Зазвичай використовується для відправки запитів до зовнішніх сервісів або для повернення відповіді клієнту.

## Call Mediator
- **Призначення**: Call Mediator також виконує відправку повідомлень, але залишається в межах потоку (proxy або API). Це дозволяє здійснювати ланцюгові виклики сервісів, передаючи результати з одного виклику до іншого.
- **Синтаксис**: 
  ```xml
  <call blocking="true">
      <endpoint>
          <address uri="http://example.com/service"/>
      </endpoint>
  </call>
  ```
- **Потік**: Call Mediator не виходить з потоку, що дозволяє продовжувати обробку повідомлень після виклику. Це особливо корисно для ланцюгових викликів, де результати одного виклику можуть бути використані в наступному.
- **Використання**: Використовується для обробки запитів, які потребують послідовних викликів до кількох сервісів.

## Основні відмінності
1. **Потік**: Send Mediator виходить з потоку, тоді як Call Mediator залишається в межах потоку.
2. **Використання**: Send Mediator зазвичай використовується для відправки запитів до зовнішніх сервісів, тоді як Call Mediator використовується для ланцюгових викликів.
3. **Обробка результатів**: Call Mediator дозволяє передавати результати між викликами, що робить його більш гнучким для складних сценаріїв.

## Висновок
Вибір між Send та Call Mediator залежить від конкретних вимог вашого проекту. Якщо вам потрібно просто відправити запит і не повертатися до обробки, використовуйте Send. Якщо ж вам потрібно виконати кілька послідовних викликів і обробити результати, Call буде кращим вибором.
